# Test Skill

## 用途

这个 Skill 用于作为标准化测试样例，帮助系统验证以下内容是否可被稳定解析：

- YAML frontmatter 是否完整且字段合法
- SKILL.md 正文结构是否清晰
- 输入信息是否能够被规范收集
- 输出内容是否符合统一格式要求
- Skill 的边界和流程说明是否齐全

它适合作为模板型、演示型、联调用的基础 Skill，不针对特定学科或学校场景。

## 首次使用时应该收集的信息

首次使用本 Skill 时，应先收集以下信息：

- 用户希望测试的目标
- 目标输出语言
- 输出内容的详细程度
- 是否需要固定格式
- 是否有平台特殊约束
- 是否需要生成示例输入与示例输出
- 是否需要保留占位符以供后续替换

如果用户没有提供完整信息，默认采用中文、标准版、Markdown 结构化输出。

## 输出规则

使用本 Skill 时，输出应遵守以下规则：

1. 输出内容应结构清晰，优先使用 Markdown 标题与项目符号。
2. 如果任务是测试解析，优先保证格式稳定，而不是追求内容复杂。
3. 若用户未指定格式，默认输出标题、用途说明、输入要求、输出结果和注意事项。
4. 不应省略关键字段、关键步骤或必要说明。
5. 不应输出与任务无关的寒暄、免责声明或冗余背景介绍。
6. 如需给出示例，示例应短小、规范、易于解析。
7. 若存在多个可选输出方案，应优先选择最稳定、最通用、最易被系统解析的方案。
8. 若用户要求严格模板化输出，应完全按照用户指定顺序与字段名生成内容。

## 边界限制

本 Skill 的边界如下：

- 不负责调用外部系统或执行真实上传行为
- 不负责验证平台后端逻辑是否通过
- 不替代具体学术、考试、邮件、展示或研究类专用 Skill
- 不生成超出用户要求的大段扩写内容
- 不在缺少关键信息时擅自编造高风险事实
- 不修改用户明确指定的字段名、格式顺序或输出约束

如果用户请求超出测试模板范围，应先保持基础结构稳定，再在允许范围内扩展内容。

## 建议流程

建议按以下流程使用本 Skill：

1. 明确本次任务是解析测试、模板测试还是输出规范测试。
2. 收集最少必要信息，包括语言、格式、详细程度和平台约束。
3. 选择最稳定的 Markdown 结构进行组织。
4. 按固定顺序生成内容，避免字段缺失或层级混乱。
5. 在输出前自检 frontmatter 是否完整、字段名是否正确、YAML 数组格式是否正确、正文模块是否齐全。
6. 输出最终结果，不附加无关解释。
