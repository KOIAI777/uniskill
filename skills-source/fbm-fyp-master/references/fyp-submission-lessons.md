# FYP Submission Lessons

本文件总结真实 FYP 终稿修订中的高频问题，供最终审查阶段优先检查。

## 1. Most Common Final-Submission Failures

- Declaration Form 缺失
- Declaration 只是自写文字，不是官方附录版式
- Acknowledgement 留有占位符
- 目录未更新，导致页码或标题与正文不一致
- 表图编号重复或跳号
- References 有 hanging indent，但排序错误
- References 格式正确，但来源质量偏弱
- `Note.` 内容有了，但样式不符合 APA
- 英文正文基本正确，但残留少量明显错词

## 2. Declaration Form Rule

- FBM 手册明确要求 `Declaration Form` 放在 `Acknowledgement` 前面
- 最终审查时，不能只检查“有没有 declaration 文本”
- 必须检查是否接近官方附录 `Appendix 2: BBA Project Declaration Form` 的版式和字段

最少应包含：

- 学校名
- 学院名
- `BBA Project (Final Year Project) – Declaration Form`
- Group #
- 两位学生姓名与学号
- supervisor
- project title
- programme
- year
- declaration 正文
- signature / date 行

## 3. TOC / Field Update Rule

如果用户修改了以下内容，必须提醒其在 Word 中更新目录字段：

- 一级或二级标题
- Appendix 标题
- Figure / Table 编号
- 分页

如果 TOC 中仍保留旧标题、旧标点或旧页码，优先判断为目录未更新。

## 4. References: Two-Layer Review

最终审查 References 时，必须分两层：

### Layer A: APA Formatting

- 字母排序
- hanging indent
- DOI / URL
- 期刊名与卷号格式
- 中文参考文献转写

### Layer B: Source Quality

- 是否为期刊论文、正式会议论文、书籍、官方报告
- 是否过度依赖博客、营销站、产品网站
- 是否用弱来源支撑核心假设

不要把“格式正确”误判成“学术质量足够”。

## 5. Notes For Tables And Figures

### Table / Figure `Note.`

推荐默认规则：

- 只有 `Note.` 标签斜体
- 后面正文正常字体
- `p` 这类统计符号单独斜体
- 缩写、样本量、显著性标记应在 `Note.` 中解释

### Which tables usually need `Note.`

强烈建议：

- correlation matrix
- ANOVA / ANCOVA
- regression table
- mediation / moderation table
- hypothesis summary table with abbreviations

可不强制：

- variable definition table
- covariate table
- experimental conditions table
- simple checklist / appendix item tables

## 6. Three-Line Table Lessons

- APA 7 三线表通常使用同一线宽
- 不要竖线
- 不要满网格边框
- 表格主体可用 `1.5` 行距提升可读性
- `Table number / title / Note.` 建议维持双倍行距
- 对复杂统计表，可在 `Model Fit` 等区块前增加一条极简横线做逻辑分组

## 7. Final Submission Checklist

提交前优先确认：

1. 官方 Declaration Form 已放在 Acknowledgement 前
2. Acknowledgement 无占位符
3. TOC 已更新
4. Figure / Table 编号连续
5. References 已排序且 hanging indent 正确
6. 弱来源未承担核心理论依据
7. `Note.` 样式与统计符号符合 APA
8. Appendix 在正文里已被提及
9. Abstract 字数符合要求
10. 导出 PDF 前再次检查分页
